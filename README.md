git status Показує стан репозиторію: які файли змінені, нові чи видалені Після редагування файлів, щоб перевірити зміни
git add <файл> Додає конкретний файл у staging area Коли хочеш закомітити лише один файл
git add . Додає всі нові та змінені файли в поточній папці Коли хочеш закомітити всі зміни у директорії
git add --all Додає всі зміни у репозиторії (нові, змінені, видалені) Коли хочеш закомітити абсолютно всі зміни
git commit -m "опис змін" Фіксує зміни у локальному репозиторії з повідомленням Після git add, щоб зберегти зміни локально
git push origin main Відправляє коміти на GitHub у гілку main Після коміту, щоб синхронізувати з віддаленим репозиторієм
git pull origin main Завантажує останні зміни з GitHub Перед роботою або якщо хтось інший оновив код
git log Показує історію комітів Коли треба переглянути, які коміти вже зроблені
git diff Показує різницю між поточними змінами та останнім комітом Коли хочеш побачити, що саме змінилось у файлах
git restore <файл> Відновлює файл до стану останнього коміту (відкидає зміни у робочій директорії) Якщо ти випадково змінив файл і хочеш повернути його назад
git restore --staged <файл> Прибирає файл зі staging area (але залишає зміни у робочій директорії) Якщо ти додав файл у staging, але передумав його комітити
